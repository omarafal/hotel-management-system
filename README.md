# Hotel Management System
A hotel management system built for the course "Software Design Patterns CSE336".

### Proposed technologies to use
- Java
- Swing (GUI)
- SQLite

### Proposed Design Patterns

## To-Do
### Manager Account
- [ ] Can sign in with a unique username and password
- [ ] Secure login with options for password recovery
- [ ] Can add, edit or delete worker profiles
- [ ] Can view a list of all hotel workers, each worker has a unique profile (name, contact information, salary, job title)
- [ ] Can view a list of all hotel residents, each resident has a unique profile (name, room details, duration of stay, service history)
- [ ] Generate reports based on the hotel's income (weekly, monthly or annual income)
- [ ] Can view a list of all hotel rooms and see their status, rooms are categorized based on room type (single, double or triple)

### Receptionist Account
- [ ] Can sign in with a unique username and password
- [ ] Can manage (edit or delete) resident details
- [ ] Can book for residents, the system calculates the cost a resident must pay at checkout

### Rooms
- [ ] Create a list of rooms with varying types and availability, price is calculated in real-time
